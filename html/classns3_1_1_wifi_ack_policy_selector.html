Detailed Description

WifiAckPolicySelector is in charge of selecting the acknowledgment policy for PSDUs containing QoS Data frames.

Destructor implementation.

This method is called by Dispose() or by the Object's destructor, whichever comes first.

Subclasses are expected to implement their real destruction code in an overridden version of this method and chain up to their parent's implementation once they are done. i.e, for simplicity, the destructor of every subclass should be empty and its content should be moved to the associated DoDispose() method.

const MacLowTransmissionParameters &  params 
)
static

Set the QoS Ack Policy for the QoS Data frames contained in the given PSDU according to the given MacLow transmission parameters.

Only single-TID A-MPDUs are supported at the moment, hence it is expected that all the QoS Data frames contained in the given PSDU have the same TID. This function is typically called by MacLow before forwarding the PSDU down to the PHY layer.

Parameters
psduthe given PSDU.
paramsthe given MacLow transmission parameters.

virtual void ns3::WifiAckPolicySelector::UpdateTxParams ( Ptr< WifiPsdu >  psdu,
MacLowTransmissionParameters &  params 
)
pure virtual

Update the transmission parameters related to the acknowledgment policy for the given PSDU.

This method is typically called by the MPDU aggregator when trying to aggregate another MPDU to the current A-MPDU. In fact, the AckPolicySelector may switch to a different acknowledgment policy when a new MPDU is aggregated to an A-MPDU.

Parameters
psduthe given PSDU.
paramsthe MacLow parameters to update.
